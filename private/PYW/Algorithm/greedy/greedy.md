- 현재 상황에서 가장 좋다고 판단되는 값을 우선적으로 취하고 이를 반복하여 가장 좋은 결과를 도출하는 방식
- greedy algorithm 적용 요건
    - 그리디 선택(Greedy Choice): 현재 선택이 이후의 선택에 영향을 미치지 않음
    - 최복 부분구조(Obtimal Substructure): 해당 문제의 최적의 솔루션이 부분 문제들의 솔루션으로 구성됨

- 예시 문제
    - <문제> 거스름 돈
        - 가장 큰 화폐 단위부터 돈을 거슬러 주는 것이 최적의 해를 보장하는 이유 -> 가지고 있는 동전 중 큰 단위가 항상 작은 단위의 배수이므로 작은 단위의 동전들을 종합해 다른 해가 나올 수 없기 때문
        - 800원을 거슬러줘야하는데 화폐 단위가 500원, 400원, 100원이라면 그리디 알고리즘 X
    - 따라서 최소한의 아이디어를 떠올리고 이것이 정당한지 검토해야함
    - <문제> 1이 될 때까지
        - 가능하면 최대한 많이 나누는 작업이 최적의 해를 보장하는 이유 -> N이 아무리 큰수여도, K로 계속 나눈다면 1을 빼는 것보다 기하급수적으로 수를 빠르게 줄일 수 있다
        - 또한 N은 항상 1에 도달할 수 있다
