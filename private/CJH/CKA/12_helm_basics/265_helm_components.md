차트가 클러스터에 적용되면 릴리스가 생성됩니다.

릴리스란 헬름 차트를 사용하여 애플리케이션을 한 번 설치하는 것을 말한다.

각 릴리스 내에서 여러 개의 리비전을 가질 수 있으며, 각 리비전은 애플리케이션의 스냅샷과 같습니다

마지막으로, 설치한 릴리스, 사용된 차트, 리비전 상태 등과 같이 클러스터에서 수행한 작업을 추적하려면 헬름이 이 데이터를 저장할 장소가 필요하다.

이 데이터를 메타데이터라고 하며, 데이터에 관한 데이터입니다.

대신 헬름은 현명하게 이 메타데이터를 쿠버네티스 클러스터에 직접 쿠버네티스 시크릿으로 저장합니다.

따라서 헬름은 이 클러스터에서 수행한 모든 작업에 대해 항상 알고 있으며 항상 메타데이터를 사용할 수 있기 때문에 모든 작업과 모든 단계를 추적할 수 있습니다.

# Helm Chart

차트는 파일의 모음으로, 쿠버네티스 클러스터에서 필요한 오브젝트 컬렉션을 생성하기 위해 헬름이 알아야 하는 모든 지침이 포함되어 있다.

# helm templating

``` value.yaml
image:
    repository: nginx
```

`{{ .Value.image.repository }}`

템플리팅 value.yaml만 수정해도 모든 배포 설정을 할 수 있다
