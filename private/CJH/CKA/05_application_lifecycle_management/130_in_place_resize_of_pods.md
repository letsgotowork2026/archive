# In-Place Pod Resizing

파드의 리소스 요구 사항을 변경하는 경우, 기본 동작은 기존 파드를 삭제한 다음 새 변경 사항이 포함된 새 파드를 스핀업합니다.

새 리소스 정의가 포함된 새 파드를 만들어야 합니다.

```
FEATURE_GATES=InPlacePodVerticalScaling=true
```

# Limitations

- CPU 및 메모리 리소스에 대해서만 작동한다.
- 파드, QoS 클래스 등 변경할 수 없습니다.
- 초기화 컨테이너와 임시 컨테이너의 크기를 조정할 수 없습니다.
- 리소스 요청 및 제한은 한 번 설정하면 이동할 수 없습니다.
- 컨테이너 메모리 제한은 사용량 이하로 줄일 수 없습니다.
- 요청으로 인해 컨테이너가 해당 상태에 놓이면 원하는 메모리 제한이 가능해질 때까지 크기 조정 상태가 계속 진행 중으로 유지됩니다.
- 현재로서는 Windows 포드의 크기를 조정할 수 없습니다.
