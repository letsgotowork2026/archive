2. Key=IT Value=Kiddie Type=NoSchedule

3. Schedule a Pod on node01 adding the correct toleration to the spec and ensure
that it lands on the correct node.

답
```
kubectl get nodes

kubectl describe node node01 | grep -i taint

kubectl taint node node01 IT=Kiddle:Noschedule

kubectl run nginx --image=nginx
kubectl run pod-toleration --image=nginx --dry-run=client -o yaml > pod-toleration.yaml

vi pod-toleration.yaml

apiVersion: v1
kind: Pod
metadata:
    creationTimestamp: null
    labels:
        run pod-toleration
    name: pod-toleration
spec:
  containers:
  - image: redis
    name: pod-toleration
  nodeName: node01
  tolerations:
  - key: "IT"
    operator: "Equal"
    value: "Kiddie"
    effect: "NoSchedule"

kubectl apply -f pod-toleration.yaml

### 16

There is a deployment named nodeport-deployment in the relative namespace.

Tasks:

1. Configure the deployment so it can be exposed using port 80 and protocol TCP name http

2. Create a new Service named nodeport-service exposing the container port 80 and TCP

3. Configure the new Service to also expose the individual pods using NodePort.

답
```
kubectl get deploy -n relative
kubectl edit deploy - n relative

ports:
- name: http
  containerPort: 80
  protocol: TCP

kubectl describe deploy -n relative | grep port

vi svc.yaml

apiVersion: v1
kind: Service
metadata:
  name: nodeport-service
  namespace: relative
spec:
  type: NodePort
  selector:
    app: nodeport-deployment
  ports:
    - port: 80
      protocol: TCP
      targetPort: 80
      nodePort: 30080
