# Symmetric Encryption

대칭 암호화

암호화, 복호화 모두 가능한 키 형태

# Asymmetric Encryption

비대칭 암호화

개인키 = Private Key

공개키 = Public Lock = 공개 잠금

`ssh-keygen`

- id_rsa = private key
- id_rsa.pub = public key = public lock

하지만 경로를 가로채면 의미가 없음

그래서 나온 것이 인증서다

키가 들어 있는 인증서를 전송합니다.

인증서를 자세히 살펴보면 실제 인증서와 비슷하지만 디지털 형식임을 알 수 있습니다.

인증서 발급자, 공개, 해당 서버의 키, 해당 서버의 위치 등에 대한 정보가 있습니다.

이것이 바로 인증 기관 또는 CAS가 필요한 이유입니다.

이들은 인증서에 서명하고 유효성을 검사할 수 있는 잘 알려진 기관입니다.

인기 있는 서비스로는 Symantec, Digicert, Comodo, Globalsign 등이 있습니다.

이 작동 방식은 앞서 생성한 키와 웹사이트의 도메인 이름을 사용하여 인증서, 서명 요청 또는 CSR을 생성하는 것입니다.
