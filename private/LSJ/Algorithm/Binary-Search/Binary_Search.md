# 이분 탐색
## 순차 탐색
: 리스트 안에 특정 데이터를 찾기 위해 앞에서부터 데이터를 하나씩 차례대로 확인
- 정렬 안된 리스트에서 사용
- 시간이 충분하면 항상 찾을 수 있음
- count() : 순차탐색 이용
- `O(n)`

## 이진 탐색
- `정렬된 리스트`에서만 사용 가능
- 탐색 범위를 절반씩 좁혀가며 데이터를 탐색
- `시작점`,`끝점`,`중간점` => 3개의 변수 사용
  
1. 중간점과 데이터 비교
2. 데이터가 중간점보다 작다면) 끝점 = 중간점 -1 => 왼쪽 확인
3. 데이터가 중간값보다 크다면) 시작점 = 중간점 +1 => 오른쪽 확인

- 확인할 때마다 확인할 원소의 개수가 1/2
- `O(logN)`
  
### 구현 방식
1. 재귀함수
   - array, target, start, end
   - array는 그대로 두고 start, end 인덱스 값만 바꿔주기
2. 반복법
   - ```while start <= end```

- 엄청 빈출 알고리즘이니 가급적 외우자
- 안보고 제대로 구현해내기 어렵다

## Tree 자료구조
### 이진 탐색 트리 BST(Binary Search Tree)
- 부모 노드보다 왼쪽 자식 노드가 작다
- 부모 노드보다 오른쪽 자식 노드가 크다

** 이진 탐색 트리에서 데이터 조회 **
1. 루트노드 방문
   - target과 비교해서 왼쪽/오른쪽 노드 방문 결정
2. 자식 노드가 없을 때까지
   - target과 비교해서 왼쪽/오른쪽 노드 방문

### 이진 탐색 문제 빠르게 입력 받기
이진 탐색 문제 유형
- 입력데이터 많거나
- 탐색 범위 넓거나
(데이터 개수 1000만개 이상, 탐색 범위가 1000억 이상)

**input() 사용시 동작속도가 느리다**
=> **sys의 readline()**

