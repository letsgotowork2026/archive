## RULE
1. 개인 학습은 `일주일`의 공부한 내용 바탕으로 정리하여 `PullRequest`로 올린다. 
2. 알고리즘 문제는 한 주에 `1개씩` 푼다.
   - 본인이 푼 문제는 간단하게 5분 설명 가능해야 한다.
   - 한 문제에 대해서 고민이 하는 시간이 30분(실버), 1시간(골드)이 넘어가지 않게 한다.
   - 그 이후에는 답을 통해 알고리즘에 익숙해지는 데 집중한다.
   - (본인이 알고리즘을 개발하는 것이 아닌 기존에 있는 것을 활용한다에 집중한다)
   - `@최준호` 주별로 문제 1개 공유
   - `Java` / `파이썬` / `C++` / 로 구현 (코테 메인 언어로 사용할 것)
   - 코테 전 계속 학습하고 익혀야할 알고리즘 리스트 (그외는 투머치이고 시간 낭비)
     - ★★★★★ (2차원 배열 문제) BFS(중요) / DFS(덜 중요)
     - ★★☆☆☆ (그래프 문제) BFS(중요) / DFS(덜 중요)
       - (코테에는 대부분 BFS로 처리 가능, DFS는 시간초과 가능성 존재)
     - ★★★★★ DP, 배낭문제
       - (배낭문제는 DP의 심화로 코테 출제 다수)
     - ★★★★☆ 누적합
     - ★★★★★ 이분탐색
     - ★★★☆☆ 두 포인터
     - ★★★★★ 다익스트라
     - ★★☆☆☆ 플로이드 워셜
     - ★★☆☆☆ 벨만포드
     - (그리디 알고리즘, 구현, 자료구조, 브루트포스, 정렬 등은 따로 공부하는 것보다)  
     - (필수 알고리즘 풀면서 자연스레 공부하는게 시간적으로 효율적이다)
